## April 2, 2026 – Progress Report

### Frontend
- Auth now supports Firebase (via `VITE_AUTH_PROVIDER=firebase`), with Firebase config in `.env`. Existing API/JWT path still available.
- Member dashboard wired to `/members/me/overview` with client-side caching (2 min stale) to reduce repeated calls.
- Payments page now lists live EMI dues and pays via API; account selector added.
- Accounts page pulls live accounts/deposits/loans with loading/empty states.
- KYC form uploads documents and submits status to backend.
- Admin dashboard pulls live counts and offers Trial Balance PDF/Excel downloads.

### Backend
- NestJS API retained; storage service supports local disk/MinIO. (Supabase storage not yet wired.)
- Firebase ID tokens can be accepted: added Firebase verification fallback and automatic user bootstrap when a Firebase-only user hits the API.
- PDF/Excel report endpoints for Trial Balance.
- Daily backup cron present in docker-compose (ignored when running without Docker).
